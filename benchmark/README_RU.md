# 🚀 Бенчмарки производительности Bloc Error Control

Данный раздел содержит инструменты для замера производительности миксина `BlocErrorControlMixin`.
Основная цель - это, чтобы продвинутая обработка ошибок и автоматическое управление токенами имели 
практически нулевой оверхед.

## 📊 Актуальные показатели (v1.0.0)
Замеры проведены на архитектуре без внешних зависимостей.


| Метрика | Результат | Описание |
| :--- | :--- | :--- |
| **Latency** | **~49.4 µs** | Задержка от `add(event)` до появления стейта |
| **Throughput** | **~69,000 ev/s** | Максимальная пропускная способность событий в секунду |
| **Memory Footprint** | **0.02 MB** | Добавочный вес 20,000 активных зон/токенов в RAM |
| **`throwIfCancelled`** | **~0.006 µs** | Стоимость одной проверки токена внутри цикла |

---

## 🏗 Что проверяют эти тесты?

1. **Latency Test**: Измеряет время создания защитной зоны (`Zone`) и токена (`EventCancelToken`) для каждого события.
2. **Throughput Test**: Проверяет, как быстро BLoC переваривает лавину из 10,000 последовательных событий.
3. **Memory Test**: Фиксирует потребление оперативной памяти при одновременном создании 20,000 контекстов выполнения.
4. **Utility Test**: Замеряет накладные расходы на вызов метода прерывания `throwIfCancelled()`.

Результаты подтверждают: использование зон и токенов стоит менее 1 микросекунды на событие, что в 16 000 раз меньше бюджета одного кадра (16.6 мс).
